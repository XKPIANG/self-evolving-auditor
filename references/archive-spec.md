# 小说存档与连续性追踪系统

## 目录结构

```
archives/
└── {书名}/
    ├── index.md             # 总索引（Wiki 目录）
    ├── log.md               # 操作日志（只追加）
    ├── book.md              # 整本书概览
    ├── characters.md        # 人物档案
    ├── hooks.md             # 钩子追踪
    ├── foreshadowing.md     # 伏笔追踪
    ├── timeline.md          # 时间线 & 关键数据
    ├── settings.md          # 设定档案（世界观/能力/势力/地理/规则）
    ├── topics/              # 主题页（编译生成，交叉引用）
    │   ├── plot-主线.md
    │   ├── plot-感情线.md
    │   └── ...
    └── chapters/
        ├── ch001.md         # 第1章档案
        ├── ch002.md         # 第2章档案
        └── ...
```

## hooks.md — 钩子追踪

追踪每个悬念/钩子的状态，防止遗漏。

```markdown
# 钩子追踪

## 未解决

| ID | 钩子内容 | 埋下章节 | 状态 |
|----|----------|----------|------|
| H001 | 神秘老者说"你的身世不简单" | 第3章 | 未解 |
| H002 | 系统提示"隐藏任务未激活" | 第7章 | 未解 |
| H003 | 李四临死前说"你不知道你父亲做了什么" | 第15章 | 未解 |

## 已解决

| ID | 钩子内容 | 埋下章节 | 解决章节 | 解决方式 |
|----|----------|----------|----------|----------|
| H000 | 主角获得神秘玉佩 | 第1章 | 第12章 | 玉佩是父亲留下的传承钥匙 |
```

## 连续性检查流程

审核新章节时，自动执行以下检查：

### 1. 数据一致性检查
```
读取 timeline.md 中的最新数据
对比新章节中的数值描述
发现不一致 → 报错

示例：
  timeline.md 记录：张三灵石余额 = 250（第8章）
  新章节（第12章）描述："张三摸了摸口袋，只剩10块钱"
  → 报错：灵石数据不一致，第8章记录余额250，第12章显示10，中间无消费记录
```

### 2. 人物状态检查
```
读取 characters.md 中的人物状态
对比新章节中的人物描写
发现矛盾 → 报错

示例：
  characters.md 记录：李四 = 死亡（第15章）
  新章节（第20章）描述："李四走了进来"
  → 报错：李四在第15章已死亡，第20章再次出场
```

### 3. 伏笔/钩子检查
```
新章节提到了某个伏笔/钩子相关的内容
→ 检查 foreshadowing.md / hooks.md 是否有记录
→ 如果解决了，更新状态
→ 如果埋下新的，记录
```

### 4. 时间线检查
```
新章节的时间是否和前面连贯
是否有时间矛盾（"三天后"但事件间隔描述不符）
```

### 5. 输出连续性报告

```markdown
# 连续性检查报告 — 第X章

## ✅ 通过
- 人物状态一致
- 时间线连贯

## ⚠️ 警告
- 张三的灵石余额未明确说明，需确认

## ❌ 错误
- [严重] 第8章记录灵石余额250，本章描述"只剩10块"，中间无消费记录
  - 第8章依据："赏金200灵石到手"
  - 本章依据："摸了摸口袋，只剩10块钱"
  - 建议：补充中间章节的消费情节，或修改本章数值
```
