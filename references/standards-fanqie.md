# 番茄小说发布标准

## 平台特征

番茄小说是字节跳动旗下免费阅读平台，用户画像：
- 主力用户：18-35岁，下沉市场为主
- 阅读习惯：快节奏、碎片化、追求爽感
- 付费模式：免费+广告，靠流量分成
- 核心指标：完读率、追读率、互动率

## 一、硬性标准（必须达标）

### 1.1 字数要求
- 单章字数：1500-4000字（推荐2000-3000）
- 总字数：签约门槛通常2万字以上
- 日更要求：签约后通常日更4000+字

### 1.2 内容红线
- [ ] 无政治敏感内容
- [ ] 无色情/暴力过度描写
- [ ] 无涉政/涉军/涉警敏感题材（除非过审）
- [ ] 无抄袭/洗稿嫌疑
- [ ] 无真实人物/事件影射
- [ ] 宗教/民族相关内容需谨慎

### 1.3 格式规范
- [ ] 章节标题格式统一（第X章 / Chapter X）
- [ ] 无乱码/特殊字符
- [ ] 段落分明，无大段堆砌
- [ ] 对话标记清晰（引号使用统一）

---

## 二、开篇标准（前3章决定生死）

番茄的推荐机制看前3章数据，前3章写不好=没有后续。

### 2.1 第1章必须做到
- [ ] 500字内出现冲突/悬念
- [ ] 1000字内展示金手指/核心设定
- [ ] 有明确的"为什么要继续看"的理由
- [ ] 不要大段描写/设定介绍/回忆

### 2.2 前3章必须做到
- [ ] 至少1个爽点
- [ ] 主角目标明确
- [ ] 核心冲突建立
- [ ] 世界观快速交代（不要单独开段解释）
- [ ] 至少1个让人想继续看的钩子

### 2.3 开篇禁忌
- ❌ 大段景物描写开头
- ❌ 主角醒来不知道在哪（太俗套，除非有新意）
- ❌ 前3章全是铺垫没有释放
- ❌ 设定堆砌（世界观说明文）
- ❌ 角色太多记不住

---

## 三、章节结构标准

### 3.1 章节开头
- [ ] 快速切入，不要慢热
- [ ] 前100字要有事件/对话/冲突
- [ ] 不要以上一章回顾开头

### 3.2 章节中段
- [ ] 推进剧情或深化角色
- [ ] 保持信息密度，不要注水
- [ ] 冲突要有升级或新信息

### 3.3 章节结尾
- [ ] 必须有钩子（悬念/反转/新信息/冲突升级）
- [ ] 不要在平淡处结束
- [ ] 让读者有理由点"下一章"

### 3.4 章节字数控制
| 类型 | 推荐字数 | 说明 |
|------|----------|------|
| 日常章 | 2000-2500 | 可以短一点 |
| 冲突章 | 2500-3500 | 需要篇幅展开 |
| 高潮章 | 3000-4000 | 允许写长 |
| 过渡章 | 1500-2000 | 尽量短 |

---

## 四、留存率优化

番茄靠广告变现，留存率=收入。关键指标：

### 4.1 完读率优化
- 章节不要太长（超过4000字完读率下降）
- 避免大段描写（读者会跳）
- 对话和动作交替（保持节奏）

### 4.2 追读率优化
- 每章结尾留钩子
- 定期释放爽点（3-5章一个）
- 主角目标要持续更新（完成一个目标就给新的）

### 4.3 互动率优化
- 制造讨论点（争议性决策/反转/新角色）
- 让读者有"我猜对了"或"我没想到"的体验
- 适当留白让读者讨论

---

## 五、题材热度参考

番茄热门题材（2025-2026）：
- 都市：赘婿、神医、兵王、重生
- 玄幻：系统流、无敌流、废柴逆袭
- 仙侠：修仙、凡人修仙
- 科幻：末日、星际
- 历史：架空、穿越
- 悬疑：诡异、规则怪谈

### 题材适配检查
- [ ] 题材是否在番茄有受众？
- [ ] 题材和文风是否匹配？（不要用严肃文学风格写赘婿文）
- [ ] 题材的核心爽点是否到位？（赘婿文的核心是打脸+身份揭露）

---

## 六、发布前检查清单

### 必查项
- [ ] 前3章有爽点
- [ ] 前3章有钩子
- [ ] 前3章角色不超过5个
- [ ] 第1章500字内有冲突
- [ ] 每章结尾有钩子
- [ ] 无内容红线问题
- [ ] 格式统一
- [ ] 无错别字/乱码
- [ ] 章节字数在合理范围

### 推荐项
- [ ] 书名吸引人（2-6字，有悬念或关键词）
- [ ] 简介有核心卖点（50字内说清楚"看什么"）
- [ ] 标签准确（影响推荐精准度）
- [ ] 封面吸引人（番茄提供模板）

---

## 发布水平评分标准（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 完全达到番茄发布标准，可直接签约上架 |
| 7-8 | 基本达标，微调后可发布 |
| 5-6 | 部分达标，需要修改开篇或结构调整 |
| 3-4 | 多项不达标，需要较大改动 |
| 1-2 | 不建议在番茄发布 |
