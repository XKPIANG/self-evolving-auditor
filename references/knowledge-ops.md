# 知识库维护操作手册

## 概述

知识库是 `knowledge/knowledge.json`，版本号记录在 `knowledge/version.txt`。
每次学习/整合/遗忘操作后，版本号 +1。

## 一、知识整合（去重 & 升级）

### 触发时机
- 每次拆解新书入库后自动运行
- 用户说"整合知识库"时手动触发

### 整合流程

```
遍历所有知识类型（rhythm_patterns / hook_patterns / logic_rules / style_patterns / anti_patterns / platform_insights）：
  1. 找相似条目（相同类型 + 相似描述/模式）
  2. 合并规则：
     - times_seen = A.times_seen + B.times_seen
     - confidence = (A.confidence * A.times_seen + B.confidence * B.times_seen) / (A.times_seen + B.times_seen)
     - tags = union(A.tags, B.tags)
     - source = union(A.source, B.source)
     - example = 保留更有描述力的那个
     - pattern/insight = 保留更精确的那个
  3. 删除被合并的条目
  4. 更新 version.txt 和 last_updated
```

### 相似度判定

两个条目"相似"的标准：
- 同一知识类型
- 描述关键词重叠 > 50%
- 或者 tags 重叠 > 60%

示例：
```
rhythm_002: "微钩子密度极高：几乎每章结尾都有新信息"  [通用, 番茄]
rhythm_new: "每章结尾必须有钩子，保持追读欲"  [番茄, 节奏]
→ 合并为 rhythm_002，times_seen++，confidence 更新
```

## 二、遗忘机制

### 触发时机
- 每分析 N 本书后自动运行（N = 已分析总数的 20%，至少 3 本）
- 用户说"遗忘检查"时手动触发

### 遗忘规则

```
遍历所有知识条目：

规则1：低频遗忘
  if times_seen == 1 且 距上次更新 > 5本书：
    → 标记为"待遗忘"（在条目中加 "pending_forget": true）
    → 如果连续2次检查都是待遗忘 → 删除

规则2：低价值遗忘
  if confidence < 0.3：
    → 直接删除（不值得保留）

规则3：平台洞察特殊处理
  platform_insights 类条目不参与自动遗忘（平台规则变化慢，保留价值高）
  但 confidence < 0.2 的仍然删除
```

### 遗忘日志

每次遗忘操作记录到 `knowledge/forget_log.json`：
```json
{
  "runs": [
    {
      "date": "2026-04-29",
      "triggered_by": "auto (book 12)",
      "forgotten": [
        {"id": "rhythm_099", "reason": "低频遗忘（连续2次待遗忘）", "original_confidence": 0.4},
        {"id": "hook_050", "reason": "低价值（confidence=0.2）"}
      ],
      "version_before": 10,
      "version_after": 11
    }
  ]
}
```

## 三、知识评估（入库决策）

### 评估标准（0-1 分）

| 维度 | 权重 | 说明 |
|------|------|------|
| 可复用性 | 0.4 | 这个模式能用在其他书上吗？ |
| 具体性 | 0.3 | "写得好看"=0分，"3章一爽点"=0.8分 |
| 反面价值 | 0.2 | 知道"什么是坏的"也有价值 |
| 独特性 | 0.1 | 和已有知识是否重复 |

### 入库阈值

```
综合评分 >= 0.6 → 入库
综合评分 0.4-0.6 → 标记为"待观察"，下次遇到同类模式再评估
综合评分 < 0.4 → 丢弃
```

### 示例

```
新知识："打脸型爽点的铺垫不宜超过5章"
- 可复用性：0.9（几乎所有打脸文通用）
- 具体性：0.8（有明确的章节数限制）
- 反面价值：0.7（知道上限在哪）
- 独特性和 hook_001 重复：0.2
- 综合：0.78 → 入库，但应整合进 hook_001
```

## 五、手动遗忘

用户说"遗忘 [关键词]"时：
```
1. 搜索所有条目，找到 pattern/rule/insight 包含关键词的条目
2. 列出匹配结果，确认删除
3. 删除确认的条目
4. 版本号 +1
```
