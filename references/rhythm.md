# 节奏分析框架

## 什么是好的节奏

节奏 = 信息密度 × 情绪波动。读者的注意力是有限资源，节奏就是分配注意力的策略。

## 节奏单元

网文的基本节奏单元是「张弛周期」：

```
铺垫（张）→ 升级（张）→ 爆发（极张）→ 舒缓（弛）→ 新铺垫
```

### 节奏过快的症状
- 每章都在打斗/冲突，读者疲劳
- 没有喘息空间，情绪麻木
- 角色没有"日常感"，像战斗机器

### 节奏过慢的症状
- 大段描写/内心独白，没有事件
- 连续多章没有冲突升级
- 读者跳章看

## 平台节奏标准

### 番茄小说

番茄读者偏好：**快节奏、强冲突、短章节**。

| 指标 | 理想值 | 警戒线 |
|------|--------|--------|
| 章节字数 | 2000-3000 | <1500 或 >4000 |
| 小高潮间隔 | 3-5章 | >7章 |
| 大高潮间隔 | 15-25章 | >30章 |
| 单章冲突数 | 1-2个 | 0个（纯铺垫章）|
| 铺垫/释放比 | 1:1 到 3:1 | >5:1 |

### 起点小说

起点读者偏好：**有深度的节奏、允许铺垫、高潮更密集**。

| 指标 | 理想值 | 警戒线 |
|------|--------|--------|
| 章节字数 | 3000-4000 | <2000 或 >6000 |
| 小高潮间隔 | 3-8章 | >10章 |
| 大高潮间隔 | 20-40章 | >50章 |
| 单章冲突数 | 1-3个 | 0个（纯铺垫章，但允许1-2章/月）|
| 铺垫/释放比 | 1:1 到 5:1 | >8:1 |
| 日常章占比 | 20-30% | >40%（节奏拖沓）|

**起点 vs 番茄节奏差异：**
- 起点允许更长的铺垫周期（世界观需要展开）
- 起点日常章有存在价值（塑造角色、展开世界观）
- 起点高潮章可以更长更细腻（读者愿意付费看详细描写）
- 起点对"注水"更敏感（付费读者对信息密度要求更高）

## 节奏评分标准（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 张弛完美，高潮间隔精准，读者几乎不可能跳章 |
| 7-8 | 节奏整体流畅，偶有1-2处拖沓或过快 |
| 5-6 | 节奏基本可用，但有明显拖沓或跳跃 |
| 3-4 | 节奏混乱，大量读者会跳章或弃书 |
| 1-2 | 没有节奏可言 |

## 节奏检查方法

### 逐章标注

对每章标注情绪值（-5 到 +5）：
- -5：极度压抑（死亡、失败）
- 0：中性（日常、过渡）
- +5：极度爽快（大胜、逆袭）

画出情绪曲线后检查：
1. 有没有连续3章以上情绪值在0附近？（拖沓）
2. 有没有连续5章以上情绪值>3？（疲劳）
3. 高潮前有没有足够的铺垫？（爆发力不足）
4. 高潮后有没有舒缓段？（情绪透支）

### 章内节奏检查

每章内部也应有小的张弛：
- 开头：快速切入（不要大段描写开头）
- 中段：冲突升级或信息推进
- 结尾：钩子（悬念/反转/新信息）

## 常见节奏问题

### 1. "流水账"节奏
症状：事件A→事件B→事件C，没有情绪起伏
原因：只记录发生了什么，没有写角色的感受和反应
修复：在事件之间加入情绪过渡

### 2. "过山车没刹车"
症状：从头爽到尾，高潮接高潮
原因：没有舒缓段，读者情绪透支
修复：高潮后加入1-2章日常/喘息

### 3. "便秘型"节奏
症状：铺垫铺了10章还没释放
原因：作者想制造悬念但拖太长
修复：铺垫中穿插小爽点，保持读者耐心

### 4. "碎片化"节奏
症状：每章都很短，事件碎裂
原因：章节切分不合理
修复：合并碎片章节，每章至少一个完整事件
