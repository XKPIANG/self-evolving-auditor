# 📖 Novel Audit — 自进化小说审核系统

> 不是静态工具，是一个**会学习的审核员**。拆解越多，审核越准。

## ✨ 核心特性

- **编译式 Wiki 追踪** — 每读一章就「编译」进 Wiki，审核第50章时不需要重读前49章
- **五维审核** — 节奏 / 爽点 / 逻辑 / 发布水平 / 设定一致性，逐章打分+批注
- **自进化知识库** — 每拆解一本书自动学习新模式，遗忘低价值知识，整合重复知识
- **双平台适配** — 番茄小说（快节奏爽文）/ 起点小说（宏大世界观），各有专属标准
- **连续性自动检查** — 人物状态、财产变动、伏笔回收、设定矛盾，自动追踪

## 🧠 工作原理

```
传统审核：每次重读全文找问题 → 没有积累
本系统：  每读一章编译进 Wiki → 持续积累，越用越准
```

| 场景 | 传统方式 | 本系统 |
|------|----------|--------|
| 审核第50章 | 重读前49章 | 查 Wiki，5秒定位 |
| 检查灵石余额 | 翻遍所有章节 | 查 timeline.md，一目了然 |
| 检查伏笔回收 | 靠记忆或手动翻 | 自动追踪 |
| 检查设定矛盾 | 人工对比 | 自动比对 |

## 📂 项目结构

```
references/          ← 基础框架（静态）
  rhythm.md            节奏分析理论
  hooks.md             爽点类型库
  logic.md             逻辑检查清单
  standards-fanqie.md  番茄小说发布标准
  standards-qidian.md  起点小说发布标准

knowledge/           ← 自进化知识库（动态）
  knowledge.json       核心知识库（学到的模式/规律）
  book_log.json        拆解记录
  version.txt          知识库版本号

archives/            ← 小说 Wiki（每本书独立目录）
  {书名}/
    index.md           Wiki 总索引
    characters.md      人物档案（状态持续追踪）
    timeline.md        时间线 & 关键数据变动
    settings.md        设定档案
    hooks.md           钩子追踪
    foreshadowing.md   伏笔追踪
    chapters/          逐章档案
    topics/            主题页（交叉引用）
```

## 🚀 使用方式

```
# 拆解样板书（学习模式）
拆解这本书 /path/to/novel.txt，书名《xxx》

# 审核小说（应用模式）
审核这段小说，发布到起点 [粘贴文本]

# 存档追踪
审核第X章 [粘贴文本]

# 知识库管理
知识库状态
```
